# jev.best — деплой, DNS, место в эстейте

Пятый стек на дроплете `64.225.108.200`. Он ничего не знает про jobhuntwow, colt-stack
и videodead и не имеет к ним доступа — общего только два объекта Docker и общий Caddy.

## 0. Где что лежит

| | |
|---|---|
| PC | `<эта папка>` |
| Дроплет | `/opt/jevbest` |
| Compose-проект | `jevbest` |
| Контейнер | `jev-web` |
| Внутренний порт | `8080` (не публикуется) |
| Публичный вход | общий Caddy `:443` → `jev-web:8080` |
| Домен | `jev.best`, `www.jev.best` → редирект на apex |
| Секреты | **их нет.** Статика, никаких ключей, `.env` не нужен |

Соседей стек касается ровно двумя вещами:

1. внешняя сеть `videodead_appnet` — через неё Caddy достаёт контейнер;
2. общий Caddyfile — туда дописывается marker-scoped блок.

## 1. Первый деплой

```bash
python jev.py deploy
```

Что происходит:

1. tar контекста (`src`, `public`, `deploy`, конфиги) — без `node_modules` и `dist`;
2. scp в `/opt/jevbest`, распаковка;
3. **образ собирается на дроплете** (`docker compose -p jevbest build`) — multi-stage,
   node:22-alpine собирает Vite, наружу идёт `caddy:2-alpine` с готовым `dist` в `/srv`;
4. `up -d --force-recreate` — **без `--remove-orphans`**, иначе снесло бы colt-promtail и ботов;
5. `deploy/fix_caddy.py` на дроплете: находит контейнер Caddy и путь к Caddyfile через
   `docker inspect`, делает бэкап, переписывает файл **на месте**, валидирует, перезагружает;
6. проверка: изнутри контейнера, по сети `appnet` и снаружи по HTTPS — все три должны отдать
   `jev-best` на `/__whoami`. «Сайт отвечает» доказательством не считается.

## 2. Что должно быть в DNS

Домен на Squarespace. Сейчас там дефолтный пресет Squarespace — его надо убрать целиком.

**Удалить:**

| Тип | Имя | Значение |
|---|---|---|
| A | @ | 198.49.23.145 |
| A | @ | 198.49.23.144 |
| A | @ | 198.185.159.145 |
| A | @ | 198.185.159.144 |
| CNAME | www | ext-sq.squarespace.com |
| HTTPS | @ | `1 . alpn="h2,http/1.1" ipv4hint="198.185…"` |

Запись **HTTPS/SVCB — обязательно**. Она содержит `ipv4hint` на серверы Squarespace, и Chrome
пойдёт по ней в обход A-записи. Пока она жива, сайт будет открываться у Squarespace, сколько
ни правь A.

**Добавить:**

| Тип | Имя | TTL | Значение |
|---|---|---|---|
| A | @ | 1 hr | `64.225.108.200` |
| A | www | 1 hr | `64.225.108.200` |

AAAA не добавляем — у дроплета IPv6 в этой схеме не используется.

Проверить, что распространилось:

```bash
python jev.py dns
```

Сертификат Caddy выпустит сам, как только `jev.best` начнёт резолвиться в дроплет —
через TLS-ALPN на уже открытом `:443`, отдельных портов открывать не нужно. Обычно
меньше минуты после первого запроса.

## 3. Ежедневно

| команда | что делает |
|---|---|
| `python jev.py deploy` | выкатить изменения |
| `python jev.py status` | отдаёт ли домен именно наш контейнер |
| `python jev.py diagnose` | кто на appnet, что в Caddyfile, что резолвится |
| `python jev.py logs` | хвост JSON-логов Caddy |
| `python jev.py down` | погасить только `jevbest` |

Локальная разработка: `npm run dev` → `http://localhost:5173`.

## 4. Почему именно так

- **Caddy внутри контейнера, а не nginx.** Один веб-сервер на весь эстейт: и на эдже,
  и в контейнере. Слушает `:8080` под uid 1000 — `CAP_NET_BIND_SERVICE` не нужен, поэтому
  `cap_drop: [ALL]` + `read_only: true` работают без исключений. В Caddyfile выключены
  `admin`, `auto_https` и `persist_config`, так что писать на диск нечего; `XDG_*` всё равно
  уведены в tmpfs `/tmp`.
- **Одна сеть.** `jev-web` подключён только к `videodead_appnet`. Контейнер в двух сетях
  заставляет Docker DNS отдавать Caddy недостижимый адрес — это уже давало плавающие 502
  на соседнем стеке.
- **Никаких published ports.** Единственная публичная поверхность — `:443` общего Caddy.
- **Сборка на дроплете.** Не тащим `dist` с PC: что собралось на сервере, то и запущено,
  один источник правды. GitHub в пути деплоя не участвует вообще.
- **`sw.js`, `index.html` и манифест отдаются с `no-cache`,** `/assets/*` и `/workbox-*`
  — с `immutable` на год. Иначе установленный PWA застрянет на старой версии.
- **`/assets/*` обслуживается без SPA-фолбэка.** Пропавший чанк должен давать 404,
  а не index.html: иначе браузер получает HTML с MIME `text/html` вместо JS и падает
  с невнятной ошибкой парсинга.
- **Проба `/__whoami` завёрнута в `handle`.** В порядке директив Caddy `try_files` идёт
  раньше `respond`, поэтому голый `respond /__whoami` успевал переписаться в `/index.html`
  и проба возвращала HTML. Ровно на этом деплой-верификация и ловится.

## 5. Что делать, если что-то не так

| Симптом | Куда смотреть |
|---|---|
| домен открывает Squarespace | осталась HTTPS/SVCB-запись или кэш DNS; `python jev.py dns` |
| 502 от Caddy | `python jev.py diagnose` — есть ли `jev-web` в списке appnet |
| старая версия в установленном PWA | hard-reload; проверь заголовок `Cache-Control` на `/sw.js` |
| Caddy не перезагрузился | `fix_caddy.py` откатывает файл сам; смотри вывод `deploy` шаг 5/5 |
| нужен бэкап Caddyfile | рядом с оригиналом лежат `Caddyfile.bak.<timestamp>` |
